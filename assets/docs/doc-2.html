Interface & Bridge Mapping

UPCI SOC/SIEM Independent Study • Tristen Dennis • Supervisor: Michael Sauer

This document maps every network interface and bridge across the lab environment: the Proxmox host bridges, each virtual machine's network devices, and the physical adapters on the Legion and Acer. It shows how each system connects to its VLAN and how mirrored traffic reaches the Security Onion sensor.

Security Onion bonds its two monitoring interfaces (net1 and net2, both on vmbr2) into a single interface, bond0, which receives mirrored traffic from the SPAN path. Its management interface (net0) is kept separate on vmbr3.

TRAFFIC & MONITORING PATH

Mirrored (SPAN) traffic from the switch enters the Proxmox host on enp6s0f0 (vmbr2) and is forwarded to Security Onion's bonded monitoring interface (bond0). The SPAN bridge has MAC learning disabled and flooding enabled so a full copy of the traffic reaches the sensor; this is applied automatically on each boot by a startup script. Management traffic is kept on a separate path (vmbr3 for the SIEM VMs, vmbr1 for the host).
